# FreePiano 现代化改造总览（2026）

更新时间：2026-03

---

# 一、项目当前状态（结论）

FreePiano 已完成核心现代化改造，当前处于：

> **工程稳定期 + 质量提升阶段**

已不再属于“架构迁移阶段”。

但需要明确：

> **现代化主链路已完成，不等于所有历史兼容路径都已彻底收口。**

当前更准确的状态是：

* 主 UI / 菜单 / 对话框 / 国际化主流程已经完成 Unicode + JSON i18n 改造
* 运行时语言切换已经可用
* 仍存在部分 ANSI / ACP 兼容路径与分散式刷新逻辑，需在当前阶段渐进式补强

---

# 二、已完成工作（当前确认状态）

## 2.1 Unicode 主链路迁移 ✅

* 工程已启用：

  * `UNICODE`
  * `_UNICODE`
* 关键 Win32 UI API 已迁移为 W 版本，例如：

  * `CreateWindowW`
  * `MessageBoxW`
  * `AppendMenuW`
  * `OPENFILENAMEW`
* 主 UI 文本加载以 `wchar_t` / 宽字符接口为主

👉 当前结论：

* **主界面链路已经 Unicode 化**
* **但项目内部仍保留部分 ANSI / ACP 兼容转换与窄字符路径**
* 因此，当前不宜再表述为“已完全摆脱 ACP / ANSI 限制”

---

## 2.2 国际化系统（i18n）重构 ✅

### 架构

* JSON 翻译文件：

  ```
  res/locales/
    ├── en.json
    ├── zh-CN.json
  ```

* 当前加载逻辑：

```
当前语言 → 英文 → 硬编码 fallback
```

* 核心接口：

```cpp
lang_load_string_w(id)
lang_format_string_w(...)
```

---

### 当前已确认特性

* 支持 `\uXXXX` Unicode
* 支持 `\u0000`（嵌入 null，用于文件过滤器）
* 支持 JSON 注释（扩展）
* 支持运行时加载语言文件

---

## 2.3 运行时语言切换 ✅（可用）

当前系统已支持：

> **无需重启即可切换语言**

已确认行为包括：

* 主菜单可重建刷新
* 主显示区可触发重绘
* 设置窗口打开时会被关闭，避免残留旧语言文本

⚠️ 需要注意：

* 运行时切换“可用”并不等于“所有窗口/控件都已实现原地刷新”
* 当前已建立统一刷新入口 `gui_refresh_all_texts()`
* 但复杂窗口仍以“关闭并重建”为主，子菜单、对话框、ListView / TreeView / ComboBox 的原地刷新能力仍可继续完善

---

## 2.4 语言配置持久化 ✅

当前已支持语言配置持久化，优先级为：

```
用户配置 > 系统语言 > 默认英文
```

当前代码实际使用的配置格式为：

```txt
ui language auto
ui language en
ui language zh-CN
```

⚠️ 注意：

* 当前实际配置文件格式不是 ini 风格的 `ui_language=...`
* 文档与代码必须以代码现状为准

---

## 2.5 GUI Unicode 改造 ✅

当前已覆盖的主要范围：

* 主窗口
* 菜单系统
* 对话框
* 设置页（包含 TreeView / ListView / ComboBox 的主要使用场景）
* 文件对话框
* VST 窗口

说明：

* 主 GUI 链路已完成 Unicode 改造
* `gui.cpp` 中用户可见文本的直接 A 版窗口文本读写已基本清空
* 但工程内部仍存在少量 ACP 转换与第三方/历史兼容边界，后续应继续渐进式清理

---

## 2.6 JSON 工具链 ✅

* 生成脚本：

  ```
  scripts/generate_locales.py
  ```

* 校验脚本：

  ```
  scripts/validate_locales.py
  ```

当前已确认：

* `en.json` 与 `zh-CN.json` 均可通过校验
* locale key 集合保持一致

---

## 2.7 系统稳定性：主流程可用 ✅

当前可以确认：

* 启动流程正常
* 语言系统可初始化
* 配置可预加载并持久化语言
* 菜单和主要 UI 可切换语言
* locale JSON 资源可被加载与校验

⚠️ 说明：

* 历史文档中“系统级回归测试已完整覆盖全部核心功能”的表述偏乐观
* 现阶段更适合表述为：

> **核心主流程可用，稳定性较好，但仍需要通过日志、调试与健壮性补强来提升可维护性**

# 四、下一阶段工作重点

## 4.1 可观测性（最高优先级）

### 目标

让系统“可被理解”，而不是黑盒。

### 需要实现

#### 日志系统

建议接口：

```cpp
fp_log_info(...)
fp_log_warn(...)
fp_log_error(...)
```

优先覆盖：

* i18n 加载成功/失败
* JSON 解析失败
* fallback 触发
* key 缺失
* 配置加载
* 关键初始化流程

输出：

* `OutputDebugStringW`（必须）
* 可选日志文件

---

## 4.2 i18n 调试能力

### Debug 开关

```cpp
#define I18N_DEBUG_SHOW_KEY
```

行为：

* 缺失翻译 → `[IDS_XXX]`
* fallback → 输出 warning

---

## 4.3 健壮性增强

重点改进：

* JSON 加载失败 → 自动 fallback，不崩溃
* 空字符串检测
* key 不存在检测
* 防止 `nullptr` 使用
* 降低窄字符/ACP 转换带来的隐藏风险

---

## 4.4 UI 刷新机制完善

虽然已支持运行时切换语言，但仍需补强：

* 菜单完全刷新（含子菜单）
* 窗口标题刷新
* 对话框文本刷新
* ListView / TreeView / ComboBox 内容刷新

建议继续围绕统一入口完善：

```cpp
gui_refresh_all_texts()
```
