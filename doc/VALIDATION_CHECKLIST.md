# FreePiano 验证与回归检查清单（2026-03）

更新时间：2026-03-24

---

# 一、文档目的

本清单用于验证以下几类改动是否正常工作：

* JSON i18n 主链路
* 运行时语言切换
* i18n fallback / missing key 调试能力
* 启动期语言覆盖能力
* 日志系统与关键可观测性
* ANSI / ACP 边界收口后的关键用户路径

原则：

* 优先检查“高风险、用户可见、最容易回归”的路径
* 保持检查步骤简洁、可重复
* 尽量覆盖中英文、非 ASCII 路径、fallback 场景

---

# 二、准备工作

## 2.1 构建前建议

建议至少准备两种构建：

* Debug
* Release

原因：

* Debug 更容易观察日志与调试显示
* Release 更接近实际用户运行环境

---

## 2.2 建议准备的文件/环境

### 语言资源

确保存在：

```txt
res/locales/en.json
res/locales/zh-CN.json
```

或运行目录下已复制：

```txt
locales/en.json
locales/zh-CN.json
```

### 配置文件

准备：

```txt
freepiano.cfg
```

可测试值：

```txt
ui language en
ui language zh-CN
ui language auto
```

### 非 ASCII 路径样本

建议准备：

* 中文目录
* 包含空格目录
* 英文目录

例如：

```txt
D:\测试目录\demo map.map
D:\Music Files\demo.fpm
```

---

# 三、日志系统验证

## 3.1 启动日志

### 检查项

启动程序后，确认日志中出现：

* 启动开始
* COM 初始化
* Config 初始化
* Language 初始化
* GUI 初始化
* Display 初始化
* Keyboard 初始化
* 主窗口显示

### 预期

日志文件：

```txt
logs/freepiano.log
```

以及 Debug 输出中可以看到类似内容：

```txt
FreePiano startup begin
COM initialized
Language subsystem initialized
GUI initialized
```

---

## 3.2 配置加载日志

### 检查项

删除或重命名 `freepiano.cfg` 后启动。

### 预期

应看到：

* 配置文件不存在 warning
* 使用默认值继续运行
* 程序不崩溃

---

## 3.3 JSON 加载日志

### 检查项

分别验证：

1. 正常存在 `en.json` / `zh-CN.json`
2. 缺失 `zh-CN.json`
3. 缺失 `en.json`
4. 人为制造 JSON 语法错误

### 预期

#### 正常情况

日志出现：

* locale 加载成功
* 字符串数量

#### 缺失 `zh-CN.json`

日志出现：

* `zh-CN` 加载失败 warning
* 但程序仍可运行

#### 缺失 `en.json`

日志出现：

* 英文 locale 缺失 warning
* JSON 翻译整体关闭
* 程序回退到硬编码文本

#### JSON 语法错误

日志出现：

* parse error
* 程序不崩溃

---

# 四、语言与 i18n 验证

## 4.1 locale 文件一致性验证

运行：

```bash
python scripts/validate_locales.py
```

### 预期

* `en.json` 作为 reference
* `zh-CN.json` 通过校验
* 无 missing/extra key 错误

---

## 4.2 启动默认语言验证

### 场景 A：配置为英文

配置：

```txt
ui language en
```

### 预期

* 启动后菜单/对话框显示英文

### 场景 B：配置为中文

配置：

```txt
ui language zh-CN
```

### 预期

* 启动后菜单/对话框显示中文

### 场景 C：配置为 auto

配置：

```txt
ui language auto
```

### 预期

* 跟随系统语言
* 系统无法识别时默认英文

---

## 4.3 启动覆盖验证

### 环境变量

```bash
set FREEPIANO_LANG=zh-CN
FreePiano.exe
```

### 命令行

```bash
FreePiano.exe --lang=en
FreePiano.exe --lang zh-CN
```

### 预期

优先级：

```txt
命令行 > 环境变量 > 配置文件 > 系统语言 > 默认英文
```

并且：

* 仅影响本次运行
* 不污染配置文件中的持久化语言
* 日志中可看到覆盖来源与最终结果

---

## 4.4 运行时语言切换验证

### 检查项

启动后，通过菜单切换语言：

* 英文 → 中文
* 中文 → 英文

### 预期

* 主菜单刷新
* 主窗口可继续正常响应
* 主显示层刷新
* 设置窗口如已打开，会被关闭
* 键位设置窗口如已打开，会被关闭
* 程序无需重启

---

# 五、i18n 调试模式验证

## 5.1 `I18N_DEBUG_SHOW_KEY`

在 `src/pch.h` 启用：

```cpp
#define I18N_DEBUG_SHOW_KEY 1
```

### 检查方式

人为删除某个 key：

* 当前语言中删除
* 英文中也删除

### 预期

UI 显示：

```txt
[IDS_XXX]
```

同时日志中出现 missing key error。

---

## 5.2 `I18N_DEBUG_SHOW_FALLBACK`

在 `src/pch.h` 启用：

```cpp
#define I18N_DEBUG_SHOW_FALLBACK 1
```

### 检查方式

仅删除当前语言中的某个 key，保留英文。

### 预期

UI 显示：

```txt
[IDS_XXX]!
```

并且日志中出现 fallback warning。

---

## 5.3 文件过滤器特殊验证

### 检查项

在调试模式开启时，执行：

* 打开 song
* 打开 VST
* 保存 song
* 导出 WAV
* 导出 MP4

### 预期

* 文件对话框仍然正常工作
* filter 字符串不应被 `[IDS_XXX]!` 破坏

---

# 六、ANSI / ACP 边界验证

## 6.1 文件对话框非 ASCII 路径验证

### 检查项

尝试：

* 从中文路径打开文件
* 保存到中文路径
* 从含空格路径打开/保存

### 预期

* 对话框可打开
* 若路径无法在 ACP 中表达，日志中会出现转换 warning/error
* 程序不应崩溃

说明：

* 当前内部仍有部分窄字符路径链路
* 因此重点是“失败可观测、行为可控”

---

## 6.2 GUI 用户可见文本链路验证

### 检查项

验证以下窗口/控件：

* 设置窗口
* keymap 编辑窗口
* key setting 窗口
* song info 对话框

### 预期

* 文本显示正常
* 中英文切换后无明显乱码
* 直接 A 版窗口文本 API 的历史路径不应再成为主要问题来源

---

## 6.3 第三方边界验证

### ASIO

检查：

* ASIO 设备枚举
* 驱动名显示

### VST

检查：

* VST 插件加载
* 编辑器窗口标题显示

### 预期

* 名称显示正常时应可直接使用
* 转换失败时应有日志
* 程序不应因名称转换失败而崩溃

---

# 七、建议优先执行的最小验证集

如果本轮只执行一轮快速验证，建议优先完成以下最小验证集。

目标：

* 用最少步骤覆盖本轮改动风险最高的路径
* 优先发现会影响正常使用、语言切换、日志观测与兼容边界的问题

## 7.1 最小验证集（推荐顺序）

### 1. 基础启动验证

操作：

* 正常启动程序一次

检查：

* 主窗口正常显示
* 无启动崩溃
* `logs/freepiano.log` 已生成
* 日志中包含启动关键步骤
* Debug/Console 构建下如带控制台，控制台可实时看到日志输出

---

### 2. locale 一致性验证

操作：

```bash
python scripts/validate_locales.py
```

检查：

* `en.json` 与 `zh-CN.json` 通过校验
* 无 missing/extra key 错误

---

### 3. 启动语言覆盖验证

操作：

```bash
FreePiano.exe --lang=en
FreePiano.exe --lang=zh-CN
```

检查：

* 启动语言与参数一致
* 日志中可看到命令行覆盖结果
* 配置文件未被临时参数污染

可选补充：

```bash
set FREEPIANO_LANG=zh-CN
FreePiano.exe
```

---

### 4. 运行时语言切换验证

操作：

* 启动程序
* 用菜单在英文/中文之间切换两次

检查：

* 菜单文本刷新
* 主窗口保持可用
* 主显示层正常刷新
* 无崩溃/卡死

---

### 5. 设置窗口切换验证

操作：

* 打开设置窗口
* 再执行运行时语言切换

检查：

* 设置窗口被关闭或安全刷新
* 主程序仍可继续操作
* 再次打开设置窗口时文案正确

---

### 6. key setting / keymap 窗口验证

操作：

* 打开 key setting 窗口
* 打开 keymap 编辑窗口
* 执行语言切换

检查：

* 窗口关闭或安全刷新符合预期
* 再次打开后文案正常
* 编辑框文本无明显乱码

---

### 7. song info 对话框验证

操作：

* 打开 song info 对话框
* 查看标题/作者/注释显示
* 如可编辑则尝试保存一次

检查：

* 文本显示正常
* 中英文切换后无明显乱码
* 保存路径不引发异常

---

### 8. 文件对话框与路径验证

操作：

* 执行打开/保存对话框
* 尽量使用：
  * 中文路径
  * 含空格路径
  * 英文路径

检查：

* 对话框正常打开
* 正常路径可工作
* 无崩溃
* 如果路径无法在当前 ACP 中表达，应在日志中可见

---

### 9. 第三方边界快速验证

操作：

* 枚举音频设备（特别是 ASIO）
* 如条件允许，加载一个 VST 插件并查看窗口标题

检查：

* 设备名/插件名显示正常或有安全兜底
* 转换失败时日志可见
* 程序不崩溃

---

## 7.2 最小验证集通过标准

如果以下项目全部通过，可认为本轮改动达到了“可进入下一阶段维护”的最低标准：

1. 程序可稳定启动和退出
2. locale 校验通过
3. 启动语言覆盖正常
4. 运行时语言切换正常
5. 设置窗口 / key setting / keymap / song info 基本可用
6. 文件对话框基本可用
7. 日志可正常生成并包含关键事件
8. ASIO / VST 边界没有引发崩溃

---

# 八、回归建议清单

每次提交后，建议至少回归以下项目：

## 最小回归集

1. 程序正常启动
2. 中英文菜单显示正常
3. 运行时切换语言正常
4. 设置窗口打开后切换语言行为正常
5. song info 对话框正常
6. key setting 窗口正常
7. locale 校验脚本通过
8. 日志文件正常生成

---

## 扩展回归集

1. ASIO / WASAPI 输出切换
2. VST 插件加载与窗口显示
3. 中文路径文件打开/保存
4. i18n 调试模式显示正确
5. 缺失 locale / 错误 locale 时程序行为可控

---

# 八、当前阶段的验收标准

如果以下条件同时满足，可认为本阶段改动达到可接受状态：

* 程序主流程稳定启动与退出
* locale JSON 可正常加载与校验
* 运行时语言切换可用
* fallback / missing key / JSON 错误均可通过日志观察
* 启动期语言覆盖能力可用
* 主要 GUI 文本无明显乱码
* 历史 ANSI / ACP 边界失败时具备日志与兜底

---

# 九、一句话总结

> 本清单的目标不是证明“所有历史问题已彻底消失”，而是确保当前阶段的关键改造已经可验证、可回归、可维护。
